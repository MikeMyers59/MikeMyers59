Web Terminology

  • Intranet: The internal Electric Boat network.
  • Internet: The World Wide Web - (External to Electric Boat's network).
  • Samba: The software used to connect computers running Microsoft Windows to Unix systems. It is what is used to map a drive letter on your local PC to the directories on the Intranet web servers.

    • Typical web development is done across multiple environments to facilitate updates and changes while only providing approved/published content to the general users of a site.

  • Development: Constantly in flux. This is where all changes and new content originates.
  • Stage: No updates are done directly in stage. It is meant to represent the Production environment for testing. Once content is determined to be stable it is moved to Stage for final review before moving to Production.
  • Production: The final product. Any updates to Production have already been applied and reviewed in Development and Stage.

Getting Started

Approvals

Have your manager email the EB Webmaster at webmaster@gdeb.com to request a web site. This request should include the purpose of the site as well as the names and badges of the employees who will create and maintain the site.

Required Accounts and Group Setup

  1. As the person responsible for the web site (Intranet individual, department, or special interest group), you must have a valid Web Development ID and password and your windows ID and web development ID must be mapped in the Samba mapping file.
  2. If you do not have what is needed in the previous step, contact the EB Webmaster webmaster@gdeb.com to have an account set up for you. Be sure to provide to us what the Web Development ID will be used for, and your Windows ID. The EB Webmaster will submit a request in ServiceNow on your behalf, that must be aproved by your manager, to have your account created.
  3. After you receive your Web development ID and password, e-mail the EB Webmaster at webmaster@gdeb.com to request a web site on the Development server. Specify the name and web development ID of the person who will be responsible for this web site development and maintenance.
  4. If you will have more than one person maintaining your web pages, you should have a Unix group created. E-mail the EB Webmaster with each employee's name, badge, and Web Development ID, as well as a group name (eight characters or less) and the EB Webmaster will submit a request in ServiceNow to have the group created.

Connecting to the Web Server

Once the web directory is created for you, and you have had your IDs placed in the Samba mapping file, you will need to map a drive to access the files on the web server.

  1. Double click on the "This PC" (Windows Explorer) icon on your desktop.
  2. Click on "Computer" in the menu.
  3. Click on Map Network Drive (from the menu).
  4. If you do not already have the W drive mapped, it is the best choice for this drive mapping. Note and remember which "Drive:" letter you are using.
  5. In the "Folder:" section, type \\samba\web.
  6. Click on the Reconnect at logon checkbox. Then click Finish button.
  7. Click the Finish button.
  8. You should see your newly mapped drive in Windows Explorer.

Now you may develop and test your web pages on the EBC Intranet web development server.

Note: To properly view your web pages you must open them by the URL. Do not open files directly from Windows Explorer as the request will not actually be processed by the web server. Example: https://www-dev.ebnet.gdeb.com/homepages/departments/604 NOT from W:\internal\dev\www-dev\homepages\departments\604.

Common Site Locations

Department pages are located at https://www-dev.ebnet.gdeb.com/homepages/departments/ and under the directory W:\internal\dev\www-dev\homepages\departments (/gdbin/www-dev/homepages/departments/).

Special interest groups are located at the following locations depending on their classifications:

  • /homepages/groups
  • /homepages/organizations
  • /homepages/projects

The webmaster will let you know your path.

Moving to Production

  1. Once you are satisfied with your web site/page in Development, email the EB Webmaster at webmaster@gdeb.com to have the Stage directory created. We will create the directory for you in Stage after we have reviewed your site in development. We will not create the directory in Stage if we find obvious problems with your site in Development.
  2. Once we have created the directory on the Stage server you will copy the contents of your site from the Development server to the Stage server.
  3. Once you are satisfied with your web site/page in Stage, email the EB Webmaster at webmaster@gdeb.com to have the Production directory created. We will create the directory for you in Production after we have reviewed your site in Stage. We will not create the directory in Production if we find obvious problems with your site in Stage. This is unlikely as you should have corrected all problems in Development.
  4. Once we have created the directory on the Production server you will copy the contents of your site from the Stage server to the Production server.
  5. The EB Webmaster will create a link from the Department or Project or Groups/Organizations page on request only after you site has been copied to Production and you have verified that the site is working properly.

If you create or are creating software for EB's use in-house, you must go to the EB Software Configuration Management Home Page and Catalog to fill out the appropriate paperwork and be Secure Code trained.

Intranet Styles and Templates

Why have a standardized framework?

Consistency

Using a standardized set of styles creates a uniform and consistent feel across the Intranet. A consistent feel helps users recognize similar elements (like buttons or navigation menu) on different sites and pages making it easier for them to get around sites they have never visited before.

Ease of Use

Having a clearly defined set of styles and templates also makes it easier on those creating sites on the Intranet. You do not have to worry about developing a basic look and feel of a website and can focus on content, not style (such as complementary colors).

Getting help is easier too. Since this has been setup by the EBIT Webmasters, it is much easier for us to troubleshoot a problem you are having or make suggestions on the best way to implement an idea you may have.

The Electric Boat Intranet Styles and Templates are designed to be basic and straightforward for use by non-IT employees.

The Intranet Styles and Templates are also easily expandable to provide the functions of a more robust framework. For more complex features check out the Advanced section.

The Intranet Styles and Templates will continue to be updated and refined. Feedback is most welcome.

Navbar Structure

The navbar is setup as an unordered list where each list item is a dropdown menu.

Each list item has the following elements:

  • An <a> element with the class 'drop.' The text for the <a> is the title of the dropdown menu.
  • A <div> element with the class specifying the width of the dropdown menu. This element will contain all of the columns for the dropdown menu.
  • An appropriate number of <div> elements with the classes specifying the width of the columns.

Below is an example dropdown menu list item with a width of 2 columns:

<li><a class="drop">2 Column</a>
	<div class="dropdown_2columns">
		
		<div class="col_2">
			<h2>2 Column</h2>
		</div>
		
		<div class="col_2">
			<p>This menu spans two columns.  This menu spans two columns.</p>
		</div>
	
	</div>
</li>

Content

All page content should be placed in a <div> element with the class 'content.' All styles are based around page content being place in a content div. Multiple content divs can be used if you would like to create separate blocks of content.

gdeb.css

Not all the styles in gdeb.css are listed below; only the classes you will most likely require to build your site. Feel free to look through the whole stylesheet if it interests you.

For those who plan on adding additional CSS to your site please note that the default box-sizing model is set to 'border-box.' This means that the width of an element includes the padding and border. It does not include the margin. This means an element with a width set to 200px, padding of 5px, and a border of 1px will still only be 200px in width on the page. This allows for easy layout of a page without having to finely adjust widths depending on varying borders and padding.

General Color Classes

  • white-text - Apply for white text
  • white - Apply for white background
  • black-text - Apply for black (technically dark gray) text
  • black - Apply for dark gray background
  • red-text - Apply for red text
  • red - Apply for red background
  • green-text - Apply for green text
  • green - Apply for green background

Being GD, blue is our primary color:

  • dark-blue - Apply for dark blue background
  • dark-blue-text - Apply for dark blue text
  • dark-blue-border - Apply for dark blue border
  • medium-blue - Apply for medium blue background
  • medium-blue-text - Apply for medium blue text
  • medium-blue-border - Apply for dark blue border
  • light-blue - Apply for light blue background
  • light-blue-text - Apply for light blue text
  • light-blue-border - Apply for dark blue border

Element Styles

  • body - The Arial font is applied to all body contents. Default font-size is set to 14px. All element font sizes are set relative to this using em
  • p - Default font-size

Content Styles

  • content - provides a centered block for page content
  • content-style - provides the styles of the content element but without the fixed width and centering

Table Styles

  • alt-rows - Applying this class makes rows in the table alternate between white and light blue

Bullet List Styles

Unordered Lists

  • no-bullets - List with no styles
  • circle-bullets - List with hollow circles
  • square-bullets - List with solid squares

Ordered Lists

  • upper-roman-bullets - List with uppercase Roman Numerals
  • lower-roman-bullets - List with lowercase Roman Numerals
  • upper-alpha-bullets - List with uppercase letters
  • lower-alpha-bullets - List with lowercase letters
  • numbered-bullets - List with numbers

Positional Classes

  • left - floats an element left
  • middle - aligns an element to be horizontally centered in the parent based on its width
  • right - floats an element right
  • centered - aligns an element to be horizontally centered in the parent with a max-width of 90% of the parent's width
  • centered-text - center aligns text
  • vertical-center - center aligns vertically
  • vertical-middle - The element is placed in the middle of the parent element
  • vertical-top - The top of the element is aligned with the top of the tallest element on the line
  • vertical-bottom - The bottom of the element is aligned with the lowest element on the line
  • vertical-subscript - Aligns the element as if it was subscript
  • vertical-superscript - Aligns the element as if it was superscript

Column Classes

  • column-container - All columns must be placed within a column-container <div> element. The column-container allows the columns to scale properly themselves without being affected by other content on a page
  • column-of-two - Place two <div>s with this class in a column-container to create two equally sized columns
  • column-of-three - Place three <div>s with this class in a column-container to create three equally sized columns
  • column-of-four - Place four <div>s with this class in a column-container to create four equally sized columns

Miscellaneous Elements

  • note-block - applies the default style for blockquote elements to any element

Dropdown Menus

The following classes are used to set the width of a dropdown menu.

  • dropdown_1column - Sets a dropdown menu width of 1 column
  • dropdown_2columns - Sets a dropdown menu width of 2 columns
  • dropdown_3columns - Sets a dropdown menu width of 3 columns
  • dropdown_4columns - Sets a dropdown menu width of 4 columns
  • dropdown_5columns - Sets a dropdown menu width of 5 columns

Dropdowns will adjust to be left or right justified automatically based on position and width of the browser window.

Column Widths

The following classes are used to set the width of individual column elements within a dropdown menu.

  • col_1 - Sets a column width of 1 column.
  • col_2 - Sets a column width of 2 columns.
  • col_3 - Sets a column width of 3 columns.
  • col_4 - Sets a column width of 4 columns.
  • col_5 - Sets a column width of 5 columns.
